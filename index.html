1.1 - IT in modern business

First thing's first: what is I.T.?

I.T. stands for information technology, which can either be the study of or the use of systems to transport and store information. The term information technology was first coined by Thomas L. Whisler and Harold J. Leavitt in a 1958 article published in the Harvard Business Review.
50 years ago, this term was rarely known, except by those who worked in large organizations, like banks and hospitals that needed to store data.

Knowledge (data) is power...

In modern business, IT operations and the world of development/coding are becoming closer and closer. With the advent of the Data Protection Act [link to GDPR part of the site here!] and the more recent GDPR legislation, IT (Cyber Security) has also played an essential part in the reinforcing integrity of businesses through the protection of their databases. These databases house essential information about customers and their journeys/interactions with businesses, which allows businesses to operate. GDPR essentially is the balance between the business need for information with the individual’s rights pertaining to their own data… more [here].

1.4 - Describe the risks that migth impact digital projects

Imagine a contract between a client in the insurance industry (let’s call them Insuracare) and the provider (let’s call these Digiteck) to redesign the Insuracare’s customer database and servicing systems. The old system was over 30 years old, and using a command-prompt interface meant that the data storage and retrieval was clunky and straining to keep up with demand, all while being rather unfriendly for the user (customer service).

In short, system outages were common, leaving the business unable to function/make a profit, and GDPR risks were very real.

Insuracare handed a huge brief with the specs of the system they required to IBM. IBM took the brief and fulfilled the job; unfortunately for them, Insuracare wasn’t too sure on certain aspects of the project, and so the latter asked for them to be changed… at great expense to both businesses. Digiteck finally decided that the project was untenable because they felt that Insuracare were being unrealistic in their expectations.

Had the proper Research and a planning/consultancy stage been done at the beginning of the project. Had the project been completed, the business would have gone from strength to strength. As it happened, the financial burden of having the project go wrong led to immediate stagnation in the insurance business as a whole; employee mobility was halted and even reversed (i.e. some managers were demoted back to customer service) and some people were made redundant.

As for the system, Insuracare would have had to make do with a system from another business who were selling the rights to it - not an ideal for an insurer whose reputation was already damaged.

In brief, digital projects require careful handling, and extensive planning.

Git

Git is is an open-source distributed version control system which can be used to track changes in your source code. It can be used to track changes in any sort of files, and is often used by groups of programmers to coordinate work on collaborative projects.
Every computer has access to a Git directory; this directory does not need to rely on network access or central servers in order to function. Git has complete full history and full tracking capabilities.

For a further explanation on how to Configure your email on Git, click here.

GitHub

GitHub is an online-based application, where you can commit your repositories (which you have prepared in Git - see above!) once you have done your coding - think of a slightly more complex ‘cloud’ like Google Drive where you upload your work.
Github offers all of the distributed version control and source code management (SCM) functionality of Git as well as adding its own features.
You use Git to ‘commit’ (or upload) your files (which contain the code) up to GitHub so you might share/collaborate more easily; you can also use Git to ‘push’ changes made to your code up there too. (for more info on how to do this, just follow the link above).
The combination of how to use these technologies has meant that I am now set up for life in regards to cross-platform work, meaning I can (first of all!) ‘save’ the work I’ve done at the Code Nation site (and the neat little Macbook they’ve allowed me access to) so I might access it from my (rather older, clunkier but equally lovely) Macbook at home.
This has wider implications for me in that I can use both Git and GitHub to start to put together a portfolio in order for me to be able to start looking at coding jobs.

    • Describe copyright and other constraints affecting websites

    Copyright refers to the protections for the creative minds behind a creation; from writers to musicians, and the designers of websites, the rights to a piece of work automatically exists, without the need to register them first; having a formal registration process in place means that it may be easier to answer questions from disputes over ownership and creation, as well as sort out any financial implications.
    Copyright and other intellectual property laws affect websites because it may restrict what you are able to publish; intellectual property and it’s nuances may also delay a project’s completion, because asking for permission to use content can also take time.
    If you have to use someone else’s content, it is good ask permission from the owners of any intellectual property. If this is not possible, it is essential to link to the original URL and to allow for content to be removed easily if you are asked to do so

    Accessibility for Websites

    Those with disabilities can often find certain websites difficult to access. This could be due to sight, deafness or sensitivities when it comes to contrast (this is not an exhaustive list of issues to take into account). Some members of the disabled community may require non-standard software or web browsers so as to be able to navigate content and applications on the internet. Here is a set of Guidelines set out by W3C. Having images on a site is definitely worthwhile, but having an image descriptor allows blind users the chance to ‘see’ the images too. Equally, if a user has problems with hearing, it is important that any sound content is properly sign posted (though many video players and applications have the option of captioning as standard). It is important therefore to think of these users when designing the navigability of the site as the starting point.
